# 比赛评委向产品经理备忘录计划
本计划旨在整合现有文档，产出一份面向大赛评委的产品经理备忘录，清晰解释前端/后端设计与开发思路。

## 目标与受众
- 受众：产品经理（非技术背景）+ 比赛评委
- 目标：用清晰、结构化的语言讲透产品定位、三端架构、关键功能流程、前后端设计与开发思路

## 信息来源（已阅读/将引用）
- README.md：产品目标、三层架构、关键流程与能力
- PROJECT_OVERVIEW.md：职责划分、系统架构、模块分工、里程碑
- THREE_AGENT_DEVELOPMENT_GUIDE.md：端到端流程、协议、API、模块职责
- ROKID_DEVELOPMENT_GUIDE.md：开发思路、会话流程、营养计算与部署
- MEAL_RECOGNITION_WHITEPAPER.md：愿景、技术创新、营养数据库
- android-phone/UI_DESIGN.md：UI 设计理念与视觉系统

## 备忘录结构（拟）
1. 项目一句话定位 + 参赛背景（sapitaljoy 全球AI/AR开发大赛）
2. 核心用户痛点与产品价值
3. 整体系统架构（眼镜端/手机端/后端）与职责边界
4. 关键业务流程（开始-持续监测-结束）与数据流
5. 前端设计思路
   - 交互原则（轻量、持续、无感）
   - 主要界面与信息层级（记录、统计、设备、档案）
   - Premium UI 设计语言摘要
6. 后端设计思路
   - 视觉识别 → 营养计算 → LLM建议
   - 营养数据库融合与动态基线策略
   - 关键 API 与会话管理
7. 开发思路与工程组织
   - 三端协作分工
   - 模块化与技术栈选择
   - 可演进路径与落地策略
8. 当前完成度与下一步计划（评委关注点）

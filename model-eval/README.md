# 食品识别模型评测工具

用于评测不同视觉大模型在食品识别任务上的表现。支持两种测试方式：
1. **网页评测工具**（推荐）- 手动在各平台测试，粘贴结果自动统计
2. **Python脚本** - 自动调用API批量测试

## 评测指标

### 基础识别能力
| 指标 | 通俗说法 | 说明 |
|------|---------|------|
| 名称准确率 | "认对了吗" | 食物名称识别正确的比例 |
| 食物数量准确率 | "认全了吗" | 图中食物数量识别正确的比例 |

### 重量估算能力
| 指标 | 通俗说法 | 说明 |
|------|---------|------|
| 重量平均误差 | "估得准吗" | 预测重量与实际重量的平均误差 |
| 重量命中率(<20%) | "猜对几个" | 重量误差<20%的样本占比 |

### 食材拆分能力（核心指标）
| 指标 | 通俗说法 | 说明 |
|------|---------|------|
| 食材召回率 | "拆得全吗" | 复合食物中识别出的真实食材比例 |
| 食材精确率 | "拆得对吗" | 识别出的食材中正确的比例 |
| 食材F1分数 | "拆分综合分" | 召回率和精确率的综合评分 |

### 属性识别能力
| 指标 | 通俗说法 | 说明 |
|------|---------|------|
| 烹饪方式准确率 | "做法对吗" | 识别烹饪方式（蒸/煮/炒/炸等）的准确率 |
| 食物类别准确率 | "分类对吗" | 识别食物类别（主食/肉类/蔬菜等）的准确率 |

### 实用性指标
| 指标 | 通俗说法 | 说明 |
|------|---------|------|
| JSON格式正确率 | "格式对吗" | 返回结果能正确解析的比例 |
| 平均响应时间 | "快不快" | 模型平均响应时间 |

## 方式一：网页评测工具（推荐）

适合没有API Key或想手动测试的用户。无需安装任何环境，直接在浏览器中运行。

### 使用步骤

1. **打开评测工具**
   - 双击打开 `eval_tool.html`
   - **工具已内置 10 组默认测试数据**（西红柿炒鸡蛋、麻婆豆腐、宫保鸡丁、清蒸鲈鱼、红烧排骨、糖醋里脊、可乐鸡翅、青椒肉丝、酸辣土豆丝、蛋炒饭），图片来源于美食天下菜谱，与菜名严格对应
   - **[新功能]** 点击 **"➕ 添加"** 按钮，可以直接在网页中录入新的测试菜品
   - 测试图片已下载至 `images/` 目录，方便您上传到模型平台

2. **开始测试**
   - 点击页面顶部的 **"复制"** 按钮，获取统一提示词
   - 打开模型平台（如 ChatGPT, 通义千问, Kimi 等）
   - **上传图片** + **粘贴提示词**
   - 等待模型生成 JSON 结果

3. **录入与分析**
   - 复制模型返回的 **JSON代码块**（不需要手动清理Markdown标记）
   - 回到评测网页，在对应图片的文本框中 **粘贴**
   - 系统会自动：
     - 解析 JSON
     - 判断名称是否正确
     - 计算重量误差
     - 分析食材召回率/精确率
     - 实时更新顶部的统计大盘
     - **[新功能]** 自动展示预测食材与真实食材的重量对比表

4. **导出报告**
   - 测试完成后，点击 **"⬇️ 导出评测结果"**
   - 生成的 CSV 包含所有详细指标，可直接用于制作图表

### 进阶：导入自定义数据

如果您想批量导入自己的测试数据，可以准备一个符合以下格式的 CSV 文件：
- **格式**：`id,image,food_name,actual_weight,food_count,ingredients,category,cooking_method`
- **食材写法**：支持 `食材名:重量` 格式（如 `米饭:150|鸡蛋:40`），多个食材用 `|` 分隔
- **导入方式**：点击工具右上角的 **"📂 导入测试数据"** 按钮选择文件即可

### 统一提示词

在各平台测试时，请使用以下统一提示词：

```
你是一名专业的营养分析助手。请仔细观察图片，精确识别图中的食物，返回JSON：
{"is_food":true,"suggestion":"建议","foods":[{"dish_name":"食材名","dish_name_cn":"食材名","category":"meal","cooking_method":"raw","ingredients":[{"name_en":"食材名","name_cn":"食材名","weight_g":100,"confidence":0.9}],"total_weight_g":100,"confidence":0.9}]}

要求：
1. 只输出JSON
2. 每种食物或菜品单独一个foods元素，不要合并
3. dish_name使用标准中文名（如：米饭、红烧肉、西兰花）
4. category: meal/snack/beverage/dessert/fruit
5. cooking_method: raw/steam/boil/braise/stir-fry/deep-fry/bake/grill
6. weight_g估算可食用部分重量(克)
7. ingredients列出所有可见食材

请根据图片识别所有可见的主要食材，并严格按照上述JSON结构输出。
```

## 方式二：Python脚本（自动化）

适合有API Key、需要批量测试的用户。

### 使用步骤

1. **准备测试数据** `test_data.csv`
   ```csv
   id,image,food_name,actual_weight,food_count,ingredients,category,cooking_method
   001,images/001.jpg,米饭,150,1,米饭:150,meal,steam
   002,images/002.jpg,东坡肉,120,1,猪肉:100|汤汁:20,meal,braise
   003,images/003.jpg,扬州炒饭,250,1,米饭:150|鸡蛋:40|火腿:30|虾仁:20|青豆:10,meal,stir-fry
   ```

2. **配置API Key**
   ```bash
   export DASHSCOPE_API_KEY="your-key"
   export OPENAI_API_KEY="your-key"
   # ...
   ```

3. **运行评测**
   ```bash
   python food_eval.py qwen2.5-vl
   python food_eval.py gpt-4.5
   ```

## 测试数据准备建议

建议准备30张测试图片，覆盖以下类别：
- 单一食材（米饭、面条、水果等）- 10张
- 复合菜品（蛋炒饭、麻辣烫、盖浇饭等）- 10张
- 多食物场景（一桌菜）- 10张

每张图片需记录：
- 食物名称
- 实际称重（克）
- 食材列表（复合菜品）
- 烹饪方式
- 食物类别
